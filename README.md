# inovhub

**Innovation Pipeline Hub** — tracking internal innovation initiatives from ideation to MVP.

## Purpose

This hub tracks things we are **actively building, evaluating, or running** as innovation projects. It uses the `innovation-pipeline` profile mode.

## What belongs here

- Tool evaluations we're conducting (e.g. Kiro, Claude Code, Windsurf)
- Internal projects and PoCs we're developing (e.g. IntelHub itself)
- Frameworks and technologies we're actively adopting for our own work (e.g. AWS Strands)
- Process experiments we're running internally (e.g. spec-driven development)

## What does NOT belong here

- Broad technology watch items we're just monitoring (those go in a tech-radar hub like `intelhub-gee`)
- Intells that only aggregate references from other hubs (those go in a central hub)

## Hub role

`team` — this hub owns its own intells directly.
